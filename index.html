Users

Two users, one shared problem: curriculum friction is invisible until it slows everyone down.

Student

Stuck on curriculum content and needs guided help, not final answers.

Admin

Non-technical and needs to see where students are struggling.

30-second framing: students need learning support; admins need signal.

Problem

Cohort programs have 100+ docs of curriculum scattered across formats. Students cannot easily find or query it, and instructors are flying blind on where students are stuck.

Option 1 Ask a classmate and hope they know.
Option 2 Google it and get the answer handed over.
The gap Neither option supports actual learning or cohort-level insight.

Solution

The system fixes both sides of the gap.

For students Guided Q&A grounded in their actual curriculum.
For admins Visibility into what is breaking down across the cohort.
For teams Usable without an engineer running reports behind the scenes.

Student Story

As a student stuck on a curriculum concept,

I want guided help based on my class materials,

So that I can keep learning without being given the final answer.

Admin Story

As an admin or instructor,

I want to see what students are asking and where they are stuck,

So that I can improve support without needing technical help.

MVP Features

Build the smallest version that proves the learning loop.

01 Student chat UI grounded in uploaded curriculum docs.
02 RAG backend with retrieval, ranking, Socratic responses, and guardrails.
03 Admin log view showing who asked what, when, and where confusion clusters.

Tech Stack

Practical stack with clear component boundaries.

Frontend React chat UI, login, role routing, admin dashboard.
Backend FastAPI for auth, chat, logs, and RAG endpoints.
Retrieval Chroma vector DB with embeddings over curriculum chunks.
Data PDFs and markdown files ingested into searchable chunks.
